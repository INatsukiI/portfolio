# done — 完了前チェックリスト

作業を完了する前に以下のステップを順番に実行し、問題があれば修正してから完了とする。

ユーザーとのやり取りは **日本語** で行うこと。

---

## ステップ 1: lint + build チェック

```bash
npm run check
```

エラーがあれば修正してから次へ進む。

---

## ステップ 2: テスト実行

```bash
npm run test
```

失敗しているテストがあれば原因を特定し、修正する。
修正後は再度 `npm run test` を実行してグリーンを確認する。

---

## ステップ 3: テスト漏れチェック

今回の作業で **新規追加・変更した** ソースファイルを確認する：

```bash
git diff --name-only HEAD
git status --short
```

以下のルールに照らし合わせてテスト漏れがないか確認する：

| 対象ファイル | 確認すること |
|---|---|
| `src/os/windows/Win*.tsx` を新規作成 | 同階層に `Win*.test.tsx` が存在するか |
| `src/os/windows/Win*.tsx` を修正 | 対応する `*.test.tsx` がある場合、テストが実装の変更を反映しているか |
| `src/os/components/**` や `src/profile.ts` を修正 | それを使っているウィンドウのテストが壊れていないか |

テスト漏れが見つかった場合は、ユーザーに「〇〇のテストがありません。書きますか？」と確認してから作成する。

---

## ステップ 4: UI 確認（UI 変更がある場合のみ）

UI に関わる変更がある場合はプレビューで目視確認する：

```
preview_start → preview_screenshot
```

---

## ステップ 5: 結果を報告
すべてのステップを完了したら、ユーザーに「lint/build/test を通し、テスト漏れも確認しました。完了です 🎉」と伝える。
